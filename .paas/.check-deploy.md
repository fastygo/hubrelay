### Check SSH access before any deploy

Use a dedicated key pair. Do not commit private keys or paste real secrets into this file.

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
ssh-keygen -t ed25519 -a 100 -f ~/.ssh/<KEY_NAME> -C "deploy-key"
chmod 600 ~/.ssh/<KEY_NAME>
chmod 644 ~/.ssh/<KEY_NAME>.pub
# Add the .pub key to the server's authorized_keys out-of-band
```

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/<KEY_NAME>
ssh -i ~/.ssh/<KEY_NAME> <DEPLOY_USER>@<SERVER_HOST>
```

### On the server after deploy

Replace `<COMPOSE_PROJECT>` with `INPUT_BOT_NAME` (for example `hub-bot`).

```bash
docker logs -f --tail 120 <COMPOSE_PROJECT>

curl -s "http://127.0.0.1:5500/healthz"

curl -s -X POST "http://127.0.0.1:5500/api/command" \
  -H "Content-Type: application/json" \
  -d '{"principal_id":"operator-local","roles":["operator"],"command":"capabilities"}'

docker compose -p "<COMPOSE_PROJECT>" -f "/opt/<COMPOSE_PROJECT>/docker-compose.yml" ps
docker ps --format '{{.Names}}'
```

### SSH tunnel from your workstation

```bash
ssh -N -L 5500:127.0.0.1:5500 -i ~/.ssh/<KEY_NAME> <DEPLOY_USER>@<SERVER_HOST>
```

Then open `http://127.0.0.1:5500` locally.

### Export deploy inputs (examples only)

Set real values in your shell or secret store; never commit them.

```bash
export INPUT_AI_API_KEY='<YOUR_AI_API_KEY>'
export INPUT_AI_BASE_URL='https://api.example.com/v1'
export INPUT_AI_MODEL='<YOUR_MODEL_ID>'

export INPUT_BOT_NAME=<COMPOSE_PROJECT>
```

### Run `paas` with a clean environment (recommended on Windows)

Shells with complex `PATH` or program-directory names can break subprocess parsing. Prefer a minimal environment:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/<KEY_NAME>
eval "$(bash ./.paas/parse_config.sh --extension bootstrap-direct --no-comments)"

env -i \
  HOME="$HOME" \
  USERPROFILE="$USERPROFILE" \
  HOMEDRIVE="$HOMEDRIVE" \
  HOMEPATH="$HOMEPATH" \
  PATH="$PATH" \
  TERM="${TERM:-xterm-256color}" \
  LANG="${LANG:-en_US.UTF-8}" \
  SSH_AUTH_SOCK="$SSH_AUTH_SOCK" \
  SSH_AGENT_PID="$SSH_AGENT_PID" \
  INPUT_BOT_NAME="$INPUT_BOT_NAME" \
  INPUT_BOT_URL="$INPUT_BOT_URL" \
  INPUT_BOT_PROFILE_ID="$INPUT_BOT_PROFILE_ID" \
  INPUT_BOT_DISPLAY_NAME="$INPUT_BOT_DISPLAY_NAME" \
  INPUT_BOT_HTTP_BIND="$INPUT_BOT_HTTP_BIND" \
  INPUT_BOT_EMAIL_ENABLED="$INPUT_BOT_EMAIL_ENABLED" \
  INPUT_BOT_EMAIL_PROVIDER="$INPUT_BOT_EMAIL_PROVIDER" \
  INPUT_BOT_EMAIL_MODE="$INPUT_BOT_EMAIL_MODE" \
  INPUT_BOT_OPENAI_ENABLED="$INPUT_BOT_OPENAI_ENABLED" \
  INPUT_AI_PROVIDER="$INPUT_AI_PROVIDER" \
  INPUT_AI_API_KEY="$INPUT_AI_API_KEY" \
  INPUT_AI_BASE_URL="$INPUT_AI_BASE_URL" \
  INPUT_AI_MODEL="$INPUT_AI_MODEL" \
  INPUT_AI_API_MODE="$INPUT_AI_API_MODE" \
  INPUT_CHAT_HISTORY="$INPUT_CHAT_HISTORY" \
  INPUT_PROXY_SESSION_ENABLED="$INPUT_PROXY_SESSION_ENABLED" \
  INPUT_PROXY_SESSION_FORCE="$INPUT_PROXY_SESSION_FORCE" \
  ./paas.exe run bootstrap-direct
```

Adjust `paas.exe` path and extension name (`bootstrap-direct`, `deploy-direct`, `deploy`) per your workflow. On Unix, call `./paas` or your packaged binary name.
